Z-PIN DELIVERY PARTNER APP - INFORMATION REFERENCE GUIDE
========================================================

This document provides detailed information about which functions, components, and values reference which files throughout the entire project.

## PROJECT STRUCTURE OVERVIEW
============================

### Core Configuration Files:
- App.tsx - Main app entry point
- app.json - Expo configuration
- package.json - Dependencies and scripts
- tsconfig.json - TypeScript configuration

### Source Code Structure:
- src/types/index.ts - TypeScript interfaces and types
- src/constants/theme.ts - Design system constants
- src/context/ - React Context providers
- src/navigation/ - Navigation configuration
- src/screens/ - Screen components
- src/components/ - Reusable components

## DEPENDENCY FLOW
==================

### Data Flow:
1. Types (index.ts) → Contexts → Screens → Components
2. Theme (theme.ts) → All UI components for styling
3. Contexts → Screens for state management
4. Navigation → Screens for routing

### Component Hierarchy:
App.tsx
├── AuthProvider (AuthContext)
├── OrderProvider (OrderContext)
└── NavigationContainer
    └── RootNavigator
        ├── LoginScreen (if not authenticated)
        ├── SignupScreen (if not authenticated)
        └── TabNavigator (if authenticated)
            ├── HomeScreen
            ├── OrdersStack
            │   ├── OrdersScreen
            │   ├── OrderDetailScreen
            │   └── NavigationScreen
            ├── EarningsScreen
            └── ProfileStack
                ├── ProfileScreen
                ├── DocumentsScreen
                ├── SettingsScreen
                ├── HelpSupportScreen
                └── BankAccountScreen

### State Management Flow:
1. AuthContext manages user authentication and profile data
2. OrderContext manages order state and actions
3. Local state in components for UI-specific data
4. Props passed down for component communication

### Styling System:
- All components import theme constants from theme.ts
- Consistent color scheme using COLORS object
- Standardized spacing using SPACING object
- Uniform border radius using BORDER_RADIUS object
- Typography system using FONTS object

## API INTEGRATION POINTS
=========================

Based on the API documentation, the following screens will integrate with backend APIs:

### Authentication Screens:
- LoginScreen.tsx → POST /api/v1/auth/partner/login
- SignupScreen.tsx → POST /api/v1/auth/partner/signup

### Home Screen:
- HomeScreen.tsx → GET /api/v1/partner/orders/available
- HomeScreen.tsx → POST /api/v1/partner/orders/:id/accept
- HomeScreen.tsx → POST /api/v1/partner/orders/:id/reject

### Orders Screen:
- OrdersScreen.tsx → GET /api/v1/partner/orders
- OrderDetailScreen.tsx → PUT /api/v1/partner/orders/:id/status

### Earnings Screen:
- EarningsScreen.tsx → GET /api/v1/partner/earnings
- EarningsScreen.tsx → GET /api/v1/partner/earnings/summary

### Profile Screens:
- ProfileScreen.tsx → GET /api/v1/partner/profile
- ProfileScreen.tsx → PUT /api/v1/partner/profile
- DocumentsScreen.tsx → GET /api/v1/partner/documents
- DocumentsScreen.tsx → POST /api/v1/partner/documents
- BankAccountScreen.tsx → GET /api/v1/partner/bank-accounts
